# 001 — Тихий выход на чистом маке + тупик Gatekeeper на Sequoia/Tahoe

**Дата:** 2026-07-19 · **Версия фикса:** 2.6

## Что случилось

Пользователь (сам автор, на новом MacBook) скачал dmg 2.5 с GitHub и не смог
запустить приложение. Наложились две независимые проблемы, из-за чего диагностика
выглядела как одна:

1. **Gatekeeper (macOS Sequoia/Tahoe).** Для ненотаризованных ad-hoc приложений
   диалог даёт только «Отмена» / «В Корзину», «правый клик → Открыть» больше не
   работает, а «Всё равно открыть» в настройках появляется не всегда. README
   советовал именно устаревший путь.
2. **Баг тихого выхода.** `LOCK_PATH` открывался внутри `~/.claude-limits-monitor`,
   но директория нигде не создавалась. На чистом маке `open()` → -1 → ветка
   «другая копия уже запущена» → молчаливый `exit(0)`. Побочно: `cache.json`
   на чистом маке тоже молча не сохранялся (`try?` глотал ошибку записи).
   Итог: даже победив Gatekeeper, пользователь получал «ничего не происходит».

## Фикс (v2.6)

- Единый `mkdir DATA_DIR` при старте до любых обращений к файлам.
- `build.sh`: `xattr -cr` + ad-hoc `codesign --force -s -` + верификация
  (раньше в dmg уезжала битая linker-signed подпись).
- `get.sh`: установка одной командой через `curl | bash` — curl-загрузки не
  получают карантин-атрибут, Gatekeeper вообще не вмешивается.
- README (ru/en): три способа установки, честная инструкция для Sequoia/Tahoe.

## Правило на будущее

1. **Любой путь к файлу в домашней папке = сначала создай директорию.** Каждый
   `open()`/`write` в `DATA_DIR`-подобные пути — только после
   `createDirectory(withIntermediateDirectories: true)`. `try?` на записи —
   запах: ошибка проглатывается молча.
2. **Тестировать установку на «чистом маке»**: перед релизом прогонять сценарий
   `mv ~/.claude-limits-monitor{,.bak}` + свежий бинарник — процесс должен жить.
   Ветка «другая копия запущена» не должна быть неотличима от «не смог стартовать»:
   в подобных ветках хотя бы логировать причину выхода.
3. **Инструкции по Gatekeeper проверять на актуальной macOS**, а не по памяти:
   поведение меняется от версии к версии (Sonoma → Sequoia → Tahoe ужесточение).
   Основной путь установки для ненотаризованных сборок — curl-инсталлер.
