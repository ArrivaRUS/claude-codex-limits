# 004 — Вид строки меню ≠ системная тема (невидимые цифры в трее)

**Дата:** 2026-07-19 · **Версия фикса:** 2.7.1

## Что случилось

Пользователь трижды сообщал «в строке меню нет процентов», при том что в панели
данные были. Причина: система в СВЕТЛОЙ теме, но строка меню визуально чёрная
(vibrancy поверх тёмных обоев). Код брал тему из `NSApp.effectiveAppearance`
(«light») и рисовал цифры цветами для светлой полосы — чёрное на чёрном.
Иконки цветные, поэтому «иконки есть, цифр нет». Периодически цифры «появлялись» —
когда фон строки меню менялся (Space/обои).

Диагноз усложнило то, что параллельно существовали НЕЗАВИСИМЫЕ причины пустого
трея (нет сессионного процента — фикс v2.7; нет входа Claude). Один симптом,
три причины: первые две починили, третья осталась — выглядело как «не починили».

## Фикс (v2.7.1)

- `isTrayDark()`: тема трея по `statusItem.button.effectiveAppearance` (кнопка
  наследует фактический вид menu bar), не по `NSApp`.
- KVO на `effectiveAppearance` кнопки → мгновенная перерисовка при смене фона
  строки меню; плюс каждый refresh-тик само-корректирует.

## Правило на будущее

1. **Для отрисовки в строке меню тему всегда брать у самой кнопки статус-айтема**,
   не у приложения/системы: menu bar может быть тёмной при светлой теме (обои).
2. **Превью не заменяет реальный экран.** В превью `dark:` задавался явно, и всё
   выглядело правильно; баг жил только в реальном трее. Финальная визуальная
   проверка — на фактической конфигурации пользователя (тема+обои), иначе она
   не закрывает гейт.
3. **Один симптом ≠ одна причина.** Если жалоба вернулась после фикса — не
   доказывать «уже починено», а искать ВТОРУЮ причину того же симптома
   (перечислить все пути, дающие симптом, и проверить каждый).
