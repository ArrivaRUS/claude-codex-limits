# 002 — Установщик Claude Code CLI не прописывает PATH

**Дата:** 2026-07-19 · **Версия фикса:** 2.7

## Что случилось

Пользователь по инструкции приложения установил Claude Code CLI
(`curl -fsSL https://claude.ai/install.sh | bash`), но `claude` в новом окне
терминала не находился: установщик положил бинарник в `~/.local/bin`, а PATH
в `~/.zshrc` не прописал. Пользователь решил, что «ничего не работает».

## Фикс (v2.7)

В пошаговую инструкцию внутри приложения добавлен шаг 2 с готовой командой
`echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc` и кнопкой копирования.

## Правило на будущее

Инструкция для пользователя обязана покрывать не только happy-path, но и
первый реальный грабель после каждого шага («установил, а команда не найдена»).
Источник таких граблей — живые сессии поддержки: каждый вопрос пользователя
«что я делаю не так?» после следования инструкции = кандидат на новый шаг
или примечание в этой инструкции.
